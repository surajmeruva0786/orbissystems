Orbis Systems proposes to design, develop, and deploy a comprehensive AI-powered Zoo & Wildlife Sanctuary Management System for Nandanvan Jungle Safari, Nava Raipur. The system replaces manual paper-based logs with a modern digital platform — enabling zookeepers, veterinary doctors, forest officers, and administrators to operate with real-time data, voice-enabled reporting in Hindi and Chhattisgarhi, and AI-assisted health monitoring. All deliverables listed in this document are included in the agreed project scope.

Complete List of Deliverables
D-01 Web Application — Frontend (React + TypeScript) Core Platform
UI & Application Shell
  • Responsive Web App — Full-featured web application built with React 18 and TypeScript, accessible from any modern browser on desktop and tablet devices.
  • Role-Based Routing — Secure, role-aware routing system that directs each user to their dedicated dashboard immediately after login, with no access to unauthorized sections.
  • Component Library — Reusable UI component library built with Tailwind CSS and Radix UI primitives — including buttons, modals, forms, tables, badges, and alert banners.
  • Framer Motion Animations — Subtle, performant micro-animations on transitions, modals, and alert states for a polished, professional user experience.
  • Mobile-Responsive Layout — Fully responsive design optimized for field workers operating on Android tablets and mobile devices in outdoor conditions.
Authentication & Access Control
  • Firebase Authentication — Secure email/password login integrated with Firebase Auth, with persistent sessions and auto-logout on inactivity.
  • Role Assignment — Admin-controlled role assignment (Zookeeper, Vet, Forest Officer, Administrator) that determines dashboard access and permissions.
D-02 Role-Based Dashboards — 4 Distinct User Interfaces Core Platform
Zookeeper Dashboard
  • Daily Animal Log Submission — Interface to submit morning (AM) and evening (PM) observation logs per animal, covering feeding status, health, behavior, and reproductive observations.
  • Voice-to-Text Log Entry — One-tap audio recording button that captures observations in Hindi or Chhattisgarhi and auto-fills the structured log form via AI transcription.
  • Photo & Media Uploads — Ability to attach photos and visual evidence directly to daily animal reports for veterinary review.
  • Log Deadline Reminders — Automated browser push notifications at 11:00 AM and 4:00 PM reminding keepers to submit pending logs, with recurring alerts for overdue submissions.
  • Per-Animal Log Status — Visual alarm indicators showing which animals have pending or overdue AM/PM logs for the current day.
  • Emergency SOS Button — Single-tap emergency alert that instantly broadcasts a push notification to all active staff across all roles for immediate incident response.
Veterinary Doctor Dashboard
  • Animal Medical History — Dedicated hospital records view showing each animal's complete medical history, past treatments, and health event timeline.
  • Medication Tracker — Interface to prescribe medications, record dosages, track ongoing treatment schedules, and mark treatments as complete.
  • AI-Structured Health Reports — Access to AI-processed health reports escalated from Zookeeper logs, with key health metrics automatically extracted and highlighted.
  • Escalated Alert Queue — Priority queue of health alerts flagged by the AI system for urgent veterinary attention.
Forest Officer Dashboard
  • Cost Analytics Dashboard — High-level statistical overview of feeding costs, species-wise expenditure breakdowns, and monthly cost trend charts.
  • Inventory Management — Stock tracking interface for meat, vegetables, grains, and critical zoo supplies with low-stock alert indicators.
  • CSV & PDF Export — One-click export of monthly reports, feeding cost summaries, and inventory records in CSV and PDF formats for official government record-keeping.
Administrator Dashboard
  • System Overview Dashboard — Bird's-eye view of zoological statistics including total animal count, active health alerts, species distribution, and daily log completion rates.
  • Staff Management — Add, edit, deactivate, or remove staff accounts and assign or change user roles across the system.
  • Animal Profile Management — Create and manage individual animal profiles including species, enclosure assignment, age, and historical health records.
D-03 Backend API Server (Flask / Python) Infrastructure
  • RESTful API Server — Production-grade Flask API server serving all frontend data requests, handling authentication, file uploads, and AI pipeline orchestration.
  • Gunicorn WSGI Deployment — API served via Gunicorn for production-level concurrency, deployed on Render cloud infrastructure for 24/7 uptime.
  • Audio Ingestion Endpoint — Dedicated API endpoint to receive voice recordings from the frontend, forward to the speech-to-text pipeline, and return structured transcripts.
  • AI Data Structuring Pipeline — Backend logic that passes transcribed text to the Groq LLM (GPT-4o mini) to extract structured health metrics — feeding status, behavior flags, medical keywords — and save them to Firestore in a standardized JSON schema.
  • Pydantic Data Models — Strict data validation models ensuring all incoming and outgoing data conforms to the defined schema, preventing corrupt records from entering the database.
  • Emergency Broadcast Handler — API endpoint that fans out SOS push notifications to all connected staff via Firebase Cloud Messaging within seconds of trigger.
  • Export Generation — Server-side PDF and CSV generation endpoints for Forest Officer reporting.
D-04 AI & Voice Intelligence Features AI / ML
  • On-Device Speech-to-Text (Android) — Primary STT using Android's built-in speech recognition for Hindi and Chhattisgarhi — zero API cost, offline-capable, covering ~80% of sessions.
  • Sarvam AI — Saaras v2 STT Fallback — Cloud-based fallback STT via Sarvam AI's Saaras v2 model for low-confidence or noisy recordings (~20% of sessions), ensuring near-100% transcription accuracy.
  • Multilingual Translation — Automatic translation of Hindi/Chhattisgarhi transcripts to English for standardized database storage and veterinary reporting.
  • AI Form Auto-Fill — LLM-powered extraction that parses free-form voice transcripts and automatically populates structured log form fields — feeding quantity, health notes, behavioral flags, urgency level.
  • Health Keyword Extraction — Automated detection of medical keywords (e.g., "limping", "not eating", "bleeding") to auto-escalate logs to the Vet's priority queue.
  • Bilingual UI Toggle — Real-time toggle to switch the entire application interface between English and Hindi without page reload.
D-05 Database, Storage & Cloud Infrastructure Infrastructure
  • Google Cloud Firestore Database — NoSQL real-time database for all application data including animal profiles, daily logs, medical records, staff accounts, and inventory — with real-time sync across all connected clients.
  • Firebase Cloud Storage — Secure cloud storage for all audio recordings and photo uploads attached to daily logs, with organised per-animal file structure.
  • Cloudinary Media CDN — Media delivery via Cloudinary for optimized image compression and fast load times on mobile field devices.
  • Firebase Security Rules — Role-level database security rules ensuring each user role can only read/write data within their permitted scope.
  • Firebase Cloud Messaging (FCM) — Push notification infrastructure for deadline reminders and Emergency SOS broadcasts.
  • Automated Data Backups — Firestore's built-in daily export functionality configured for disaster recovery and audit trail compliance.
D-06 Android Mobile Application (Field-Ready) Mobile
  • Progressive Web App (PWA) — Mobile-optimized PWA installable on Android devices, providing a near-native app experience without requiring Google Play Store distribution.
  • On-Device Voice Recording — Native Android microphone integration for voice log recording directly in the field, even in areas with intermittent connectivity.
  • Touch-Optimized UI — Large tap targets, swipe-friendly navigation, and high-contrast visual design optimized for outdoor sunlight conditions.
  • Push Notification Support — FCM-powered push alerts for log deadlines (11 AM / 4 PM) and emergency SOS events delivered directly to keeper and vet devices.
  • Camera Integration — Direct camera access for attaching photos to animal health reports without leaving the application.
D-07 Deployment, Hosting & DevOps Deployment
  • Frontend Deployment — Vercel — Web application hosted on Vercel's global CDN with automatic HTTPS, custom domain configuration, and zero-downtime deployments.
  • Backend Deployment — Render — Flask API server deployed on Render's Web Service with auto-scaling, health checks, and automatic restarts on failure.
  • Environment Configuration — Secure environment variable management for all API keys (Groq, Sarvam AI, Cloudinary, Firebase) using platform-native secret stores.
  • CI/CD Pipeline — Automated build and deployment pipeline triggered on every code push, ensuring rapid and reliable updates to the live system.
  • Custom Domain Setup — Assistance with custom domain (e.g., junglesafari.orbissystems.co.in) DNS configuration and SSL certificate provisioning.
  • Deployment Documentation — Complete step-by-step deployment guide (DEPLOYMENT_GUIDE.md) covering backend, frontend, and environment setup for the client's internal IT team.
D-08 Documentation, Training & Handover Support
  • Technical Documentation — Complete system README, API endpoint reference, data schema documentation, and architecture diagram for internal IT staff.
  • User Manuals — Role-specific user guides (in English and Hindi) for Zookeepers, Vet Doctors, Forest Officers, and Administrators.
  • Onboarding & Training Session — Live or recorded walkthrough session with key staff members to demonstrate all features and answer operational questions.
  • Source Code Handover — Full, commented source code repository transferred to the client upon project completion and final payment.
  • 30-Day Post-Launch Support — One month of bug-fix support post-deployment at no additional charge, covering any issues arising from production use.
